Projects

BrainTap


Have you ever wanted to control your laptop or computer with your mind? Well, using a Melon EEG headband, my team and I were able to do precisely that. We mapped brainwaves obtained from the Melon EEG headband, and voice data obtained from a Microsoft Kinect, to create a voice and brain controlled laptop. We mapped different brainwaves to different keys on the keyboard, and certain words to defined macros, so that we could do a host of things without ever touching our laptop. More information can be found on our project page here.

Home.ly


Sometimes getting up to turn off the light when you're comfortable can be a pain. Keeping this in mind, my team and I set out to create a method by which we could control systems in our house without ever getting up. We combined Myo armbands with Samsung Smart Things to create a gesture controlled home. Using this you could control individual lightbulbs, locks, lights, speakers, etc. However, we still thought this wasn't lazy enough, so we decided to make 'modes'. Some modes included 'party' mode, which set the lights and sound system for a party, and 'lockdown' mode which locked your doors and switched off non-essential lights. More information can be found on our project page here.

SmartSecurity


I take security very seriously, and sometimes a simple, yet effective security system is a necessity. So we set out to make a simple security system in 24 hours. Using a simple Node.js HTTP server, and javascript wrapper for a raspberry pi, we were able to create a two factor authentication security system, using a fingerprint as well as a visual indicator to open a lock. As a later addition, we tried to add in a Myo armband, but thought that would be making it too complex.

A Simple Shell


Made as part of my Operating Systems class, the simple shell had marginally better performance than the native BASH shell. Implemented fully in C, with multithreading and multiprocess handling capabilities, this was my first introduction to low-level programming.

HTTP and TCP servers


As a part of my Networks class, I implemented HTTP as well as TCP servers and clients. The HTTP servers were configured to handle multiple clients and had standard behavior. The TCP server used RENO congestion control, and could handle multiple connections as well. This was my first introduction to socket-level programming and network programming.
